# Reference Splitting Guidance

Move these into `references/` or `assets/`:

- Long examples and templates
- Error handling sections
- Step-by-step command blocks
- Large tables and checklists
- Repeated guidelines across skills

Keep in SKILL.md:

- Purpose and when to use
- Minimal workflow steps
- References (load order)
- Only essential constraints

When splitting:

- Use short, named files by topic
- Keep references discoverable in SKILL.md
- Avoid circular references
